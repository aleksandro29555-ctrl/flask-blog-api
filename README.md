# Flask Blog API

Учебный REST API для ведения блога, реализованный на Flask 3.

В проекте реализованы две сущности:

- **User** — пользователь
- **Twit** — пост блога

API предоставляет возможность:

- создавать пользователей
- создавать посты от имени пользователя
- получать список постов
- изменять пост
- удалять пост
- получать все посты конкретного пользователя

Хранилище — in-memory (без базы данных).  
Все данные сбрасываются при перезапуске сервера.

Проект выполнен в рамках учебного задания и демонстрирует базовые принципы работы с REST API на Flask 3.

---

## Требования

- Python 3.10+
- Flask 3.x

---

## Установка и запуск

1. Установить зависимости:

```bash
pip install -r requirements.txt
python app.py
```

По умолчанию сервер стартует по адресу: http://127.0.0.1:5050

## Пользователи (User)

Создать пользователя: POST /user, в теле запроса: {"username": "@username"}
Получить список пользователей: GET /user
Получить список всех постов пользователя: GET /user/USER_ID/twits, где USER_ID - ID существующего пользователя.

## Посты (Twit)

Создать пост: POST /twit, в теле запроса: {"body": "text", "author_id": USER_ID}, где USER_ID - ID существующего пользователя.
Получить список постов: GET /twit
Изменить текст поста: PUT /twit/TWIT_ID, в теле запроса {"body": "new_text"}, где TWIT_ID - ID существующего поста.
Удалить пост: DELETE /twit/TWIT_ID, где TWIT_ID - ID существующего поста.

## Примечания
	-	Все данные хранятся в памяти приложения и удаляются при перезапуске сервера.
	-	Проект предназначен для учебных целей и не использует базу данных или систему авторизации.
	-	Сущность Twit используется как пост блога в рамках задания.
